#### 1. 识别干系人 (启动过程组)
定期识别项目干系人，分析和记录他们的利益、参与度、相互依赖性、影响力和对项目成功的潜在影响。

#### 2. 规划干系人参与 (规划过程组)

## 17.1 管理基础

### 17.1.1 管理的重要性
*   **定义**：每个项目都有干系人，他们会受到项目积极或消极的影响，或者能对项目施加积极或消极的影响。
*   **目标**：干系人满意度应作为项目目标加以识别和管理。
*   **关键点**：重视所有干系人，保持持续沟通，理解需求和期望，处理问题，管理利益冲突，促进参与。
*   **迭代开展时机**：
    1.  项目进入其生命周期的不同阶段。
    2.  当前干系人不再与项目有关，或出现新的干系人。
    3.  组织内部或更大领域的干系人群体发生重大变化。

![](./images/第17章_项目干系人管理/pic_002.jpg)

### 17.1.2 管理新实践
1.  **识别所有干系人**：而非仅在限定范围内。
2.  **全员参与**：确保所有团队成员都涉及引导干系人参与的活动。
3.  **定期审查**：可与单个项目风险的审查工作并行开展。
4.  **应用“共创”概念**：咨询受影响最大的干系人，视其为合作伙伴。
5.  **关注价值**：关注干系人有效参与程度的正面与负面价值。

## 17.3 识别干系人

本过程通常在编制和批准项目章程之前或同时首次开展。需在项目生命周期中**定期重复开展**。

![](./images/第17章_项目干系人管理/pic_006.jpg)

### 17.3.1 输入
1.  **立项管理文件**。
2.  **项目章程**。
3.  **项目管理计划**（首次识别时不存在，后续可参考沟通管理计划、干系人参与计划）。
4.  **项目文件**（需求文件、问题日志、变更日志）。
5.  **协议**。
6.  **EEF & OPA**。

### 17.3.2 工具与技术

#### 1. 数据收集
*   **问卷和调查**。
*   **头脑风暴**：向小组征求意见。
*   **头脑写作**：头脑风暴的改良形式，给参与者单独思考时间。

#### 2. 数据分析
*   **干系人分析**：产生干系人清单和各种信息（岗位、角色、利害关系、期望、态度等）。
    *   利害关系组合：①兴趣；②权利（合法或道德）；③所有权；④知识；⑤贡献。
*   **文件分析**。

#### 3. 数据表现：干系人映射分析和表现 (重点)

**(1) 权力/利益方格、权力/影响方格、作用/影响方格**
*   **权力**：职权级别。
*   **利益**：对项目成果的关心程度。
*   **影响**：主动参与或施加影响的意愿。
*   **作用**：施加影响后改变项目的能力。

**权力/利益方格管理策略：**
*   **B区 (权高利高)**：**重点管理**（如重要客户、主管领导）。
*   **C区 (权低利高)**：**随时告知**（需保护其利益）。
*   **A区 (权高利低)**：**令其满意**。
*   **D区 (权低利低)**：**监督**（花最少精力）。

> **口诀**：权高利低让他爽，权低利高常告知，权高利高重管理，权低利低仅监督。

![](./images/第17章_项目干系人管理/pic_008.jpg)

**(2) 干系人立方体**
将方格模型中的要素组合成三维模型，便于分析。
![](./images/第17章_项目干系人管理/pic_009.jpg)

**(3) 凸显模型**
适用于复杂的干系人群体。基于三个维度分类：
1.  **权力**（职权或影响力）。
2.  **紧迫性**（时间约束或重大利益）。
3.  **合法性**（参与的适当性）。
*注：也可以用邻近性取代合法性。*
![](./images/第17章_项目干系人管理/pic_010.jpg)

**(4) 影响方向**
*   **向上**：高级管理层、客户、发起人。
*   **向下**：团队成员、专家。
*   **向外**：供应商、政府、公众。
*   **横向**：同级项目经理、中层管理人员。

**(5) 优先级排序**
适用于有大量干系人或关系复杂的情况。

![](./images/第17章_项目干系人管理/pic_011.jpg)
![](./images/第17章_项目干系人管理/pic_012.jpg)

### 17.3.3 输出
**干系人登记册**：
*   **身份信息**：姓名、职位、地点、联系方式、角色。
*   **评估信息**：需求、期望、影响力、关键阶段。
*   **干系人分类**：内部/外部、作用/影响/权力/利益等。

## 17.4 规划干系人参与

应在项目生命周期**早期**制定，并**定期审查和更新**。
**触发更新的情况**：新阶段开始、组织结构变化、干系人变化、其他过程输出导致需要重新审查。

![](./images/第17章_项目干系人管理/pic_014.jpg)
![](./images/第17章_项目干系人管理/pic_015.jpg)

### 17.4.1 输入
项目章程、项目管理计划、项目文件、协议、EEF、OPA。
![](./images/第17章_项目干系人管理/pic_016.jpg)

### 17.4.2 工具与技术
1.  **专家判断**。
2.  **数据收集**：**标杆对照**（与引领级组织比较）。
3.  **数据分析**：
    *   **假设条件和制约因素分析**。
    *   **根本原因分析**：识别导致某种支持水平的根本原因。
4.  **决策**：**优先级排序或分级**。
5.  **数据表现** (重点)：
    *   **思维导图**。
    *   **干系人参与度评估矩阵**：比较当前参与水平 (C) 与期望参与水平 (D)。

**参与水平分类：**
1.  **不了解型**：不知道项目。
2.  **抵制型**：知道但抵制变更。
3.  **中立型**：了解，不支持也不反对。
4.  **支持型**：了解并支持。
5.  **领导型**：了解并积极参与，确保成功。

**矩阵示例：**
| 干系人 | 不知晓 | 抵制 | 中立 | 支持 | 领导 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 干系人1 | C | | | D | |
| 干系人2 | | | C | D | |
| 干系人3 | | | | D, C | |

*   **C (Current)**：当前参与水平。
*   **D (Desired)**：期望参与水平。
*   **目标**：填补 C 与 D 之间的差距。

6.  **会议**。

![](./images/第17章_项目干系人管理/pic_017.jpg)

### 17.4.3 输出
**干系人参与计划**：确定用于促进干系人有效参与决策和执行的策略和行动。

![](./images/第17章_项目干系人管理/pic_018.jpg)

## 17.5 管理干系人参与

**核心活动**：引导参与、管理期望、处理风险和问题、澄清和解决问题。

![](./images/第17章_项目干系人管理/pic_019.jpg)
![](./images/第17章_项目干系人管理/pic_020.jpg)

### 17.5.1 输入
项目管理计划、项目文件（问题日志、变更日志等）、EEF、OPA。

### 17.5.2 工具与技术
1.  **专家判断**。
2.  **沟通技能**：反馈。
3.  **人际关系与团队技能**：
    *   **冲突管理**：及时解决。
    *   **文化意识**：考虑文化差异。
    *   **谈判**：获得支持或达成协议。
    *   **观察和交谈**：了解工作和态度。
    *   **政策意识**：了解权力关系。
4.  **基本规则**：明确行为规范。
5.  **会议**。

![](./images/第17章_项目干系人管理/pic_021.jpg)

### 17.5.3 输出
变更请求、项目管理计划更新、项目文件更新。
![](./images/第17章_项目干系人管理/pic_022.jpg)

## 17.6 监督干系人参与

**主要作用**：随着项目进展和环境变化，维持或提升干系人参与活动的效率和效果。

![](./images/第17章_项目干系人管理/pic_023.jpg)

### 17.6.1 输入
项目管理计划、项目文件、工作绩效数据、EEF、OPA。

### 17.6.2 工具与技术
1.  **数据分析**：
    *   **备选方案分析**：评估应对偏差的方案。
    *   **根本原因分析**：确定未达预期效果的原因。
    *   **干系人分析**。
2.  **决策**：多标准决策分析、投票。
3.  **数据表现**：干系人参与度评估矩阵（跟踪变化）。
4.  **沟通技能**：反馈、演示。
5.  **人际关系与团队技能**：积极倾听、文化意识、领导力、人际交往、政策意识。
6.  **会议**。

![](./images/第17章_项目干系人管理/pic_024.jpg)

### 17.6.3 输出
工作绩效信息、变更请求、更新。

![](./images/第17章_项目干系人管理/pic_025.jpg)

## 本章练习与真题

**练习题：**

(1) ( ) 不属于项目干系人管理的输入。
A. 干系人管理计划
B. 干系人沟通需求
C. 变更日志
D. 变更请求
**【答案】D**
> 解析：变更请求是输出，不是输入。

(2) 识别干系人的依据不包括 ( )。
A. 可行性研究报告
B. 项目章程
C. 风险登记册
D. 问题日志
**【答案】C**
> 解析：识别干系人的输出是干系人登记册，风险登记册是更新的内容，或者是后续过程的输入，但在初次识别时，风险登记册可能尚未建立（或作为更新对象）。*注：按照PMBOK指南，识别干系人的输入包括项目文件（需求文件、问题日志、变更日志），不直接包含风险登记册。*

![](./images/第17章_项目干系人管理/pic_026.jpg)

(3) 监督干系人参与过程的作用是 ( )。
A. 维持或提升干系人参与活动的效率和效果
B. 提高干系人对项目的支持度，并尽可能降低干系人对项目的抵制
C. 提供与干系人进行有效互动的可行计划
D. 项目团队能够建立对每个干系人或干系人群体的适度关注
**【答案】A**
> 解析：B是管理的干系人参与的作用，C是规划的作用，D是识别的作用。

![](./images/第17章_项目干系人管理/pic_027.jpg)

**判断题：**
(1) 干系人管理过程即识别干系人、规划干系人参与和管理干系人参与的过程。 (×)
> 解析：还包括监督干系人参与。
(2) 干系人包括项目团队成员。 (√)
(3) 识别干系人应在项目初期开展且仅开展一次，尽可能识别所有干系人。 (×)
> 解析：应在整个项目期间定期开展。
