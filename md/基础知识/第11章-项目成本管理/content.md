## 11.1 管理基础

### 11.1.1 重要性和意义
项目成本管理重点关注完成项目活动所需资源的成本，同时也考虑项目决策对产品全生命周期成本（使用、维护、支持）的影响。
**成本失控原因**：认识不足、制度不健全、方法问题、技术制约、需求管理不当。

### 11.1.2 相关术语和定义 (重点)

#### 1. 成本类型
*   **可变成本 (Variable Cost)**：随生产量/工作量变化（如原材料、燃料）。
*   **固定成本 (Fixed Cost)**：不随生产量变化（如设备折旧、房租）。
*   **直接成本 (Direct Cost)**：直接归属于项目工作（如团队工资、差旅费、项目专用物料）。
*   **间接成本 (Indirect Cost)**：由几个项目共同分担（如公司管理费、水电费、保安费）。
*   **机会成本 (Opportunity Cost)**：选择一种机会而放弃的其他最佳替代机会的收益。
*   **沉没成本 (Sunk Cost)**：已发生且无法改变的成本。**决策时不应考虑沉没成本**。

#### 2. 储备分析 (Reserve Analysis)
*   **应急储备 (Contingency Reserve)**：
    *   应对**“已知-未知”**风险（已识别但未发生）。
    *   包含在**成本基准**中。
    *   使用**不需要**走变更流程（项目经理有权使用）。
*   **管理储备 (Management Reserve)**：
    *   应对**“未知-未知”**风险（不可预见）。
    *   **不包含**在成本基准中，但属于项目总预算。
    *   使用**需要**走变更流程（需高层批准）。

**关系图示**：
![](./images/第11章_项目成本管理/pic_006.jpg)
> **总资金需求** = 成本基准 + 管理储备
> **成本基准** = 工作包成本估算 + 应急储备

#### 3. 成本基准
经批准的、按时间段分配的项目预算。不包括管理储备。用于度量和监督项目的实际执行成本。通常以 **S曲线** 形式呈现。

### 11.1.3 管理新实践
**挣得进度 (Earned Schedule, ES)**：
*   **ES**：挣得进度，是EVM的延伸。
*   **AT**：实际时间。
*   **新指标**：
    *   进度偏差：$TV = ES - AT$ （大于0表示提前）。
    *   进度绩效指数：$SPI = ES / AT$。

## 11.4 估算成本

**作用**：确定项目所需的资金。
**注意**：估算应覆盖全部资源（人工、材料、设备、服务、设施、融资成本、通货膨胀补贴等）。随着项目进展，估算准确性逐步提高（粗略量级 $\rightarrow$ 预算估算 $\rightarrow$ 确定性估算）。

![](./images/第11章_项目成本管理/pic_016.jpg)
![](./images/第11章_项目成本管理/pic_017.jpg)

### 11.4.1 输入
项目管理计划、项目文件、EEF、OPA。
![](./images/第11章_项目成本管理/pic_018.jpg)

### 11.4.2 工具与技术 (重点)
1.  **专家判断**。
2.  **类比估算**：利用历史数据，成本低、速度快，但准确性较低。
3.  **参数估算**：利用统计关系（如：每平方米造价 $\times$ 平方米数）。准确性取决于模型成熟度和数据可靠性。
4.  **自下而上估算**：最准确，但成本高、耗时长。
5.  **三点估算** (PERT)：
    *   **三角分布**：$C_E = (C_O + C_M + C_P) / 3$
    *   **贝塔分布**：$C_E = (C_O + 4C_M + C_P) / 6$
    *   *(注：$C_O$ 乐观, $C_M$ 最可能, $C_P$ 悲观)*
6.  **数据分析**：
    *   **储备分析**：计算应急储备。
    *   **质量成本** (COQ)。
7.  **PMIS**。
8.  **决策** (投票)。

![](./images/第11章_项目成本管理/pic_019.jpg)
![](./images/第11章_项目成本管理/pic_020.jpg)

### 11.4.3 输出
1.  **成本估算**：包括所有资源的成本及应急储备。
2.  **估算依据**。
3.  项目文件更新。

### 11.5.1 输入

### 11.5.2 工具与技术
1.  **专家判断**。
2.  **成本汇总**：工作包 $\rightarrow$ 控制账户 $\rightarrow$ 整个项目。

2.  **项目资金需求**：总资金需求 = 成本基准 + 管理储备。资金通常以**阶梯状**增量投入。

## 11.6 控制成本

**作用**：保持对成本基准的维护。需在整个项目期间开展。

![](./images/第11章_项目成本管理/pic_030.jpg)
![](./images/第11章_项目成本管理/pic_031.jpg)

### 11.6.1 输入
1.  项目管理计划。
2.  项目资金需求。
3.  项目文件。
4.  **工作绩效数据**。
5.  OPA。

### 11.6.2 工具与技术 (核心计算考点)

#### 1. 挣值分析 (EVM)
*   **PV (计划价值)**：计划要做的工作的预算价值。
*   **EV (挣值)**：实际完成工作的预算价值。
*   **AC (实际成本)**：实际花费的成本。

**偏差分析**：
*   **进度偏差 SV** = $EV - PV$ （$>0$ 提前；$<0$ 滞后）
*   **成本偏差 CV** = $EV - AC$ （$>0$ 节约；$<0$ 超支）

**绩效指数**：
*   **进度绩效指数 SPI** = $EV / PV$ （$>1$ 提前；$<1$ 滞后）
*   **成本绩效指数 CPI** = $EV / AC$ （$>1$ 节约；$<1$ 超支）

#### 2. 预测 (Forecasting)
*   **BAC**：完工预算（成本基准的总和）。
*   **EAC**：完工估算（预测最终总成本）。
*   **ETC**：完工尚需估算（剩余工作还需要多少钱）。

**EAC 计算公式**：
1.  **典型偏差**（当前偏差代表未来趋势）：
    $$EAC = BAC / CPI$$
2.  **非典型偏差**（当前偏差不再发生，未来按计划进行）：
    $$EAC = AC + (BAC - EV)$$
3.  **同时考虑 SPI 和 CPI**（进度和成本双重影响）：
    $$EAC = AC + [(BAC - EV) / (CPI \times SPI)]$$

#### 3. 完工尚需绩效指数 (TCPI)
为了实现特定的管理目标（如BAC或EAC），剩余工作必须达到的成本绩效指标。
*   按原预算 BAC 完成：
    $$TCPI = (BAC - EV) / (BAC - AC)$$
*   按新估算 EAC 完成：
    $$TCPI = (BAC - EV) / (EAC - AC)$$
*   **判断**：$TCPI > 1$，这就很难完成；$TCPI < 1$，这就很容易完成。

### 11.6.3 输出
1.  **工作绩效信息**（CV, SV, CPI, SPI, TCPI, VAC等）。
2.  **成本预测** (EAC)。
3.  变更请求。
4.  项目管理计划/文件更新。

C. 绩效测量规划
D. 准确度
**【答案】D** （注：控制临界值通常指偏差允许范围，此处根据选项，准确度更接近对估算区间的要求，但根据PMBOK定义，临界值更为准确。若仅在选项中选，准确度涉及区间范围。通常 $\pm 5\%$ 描述的是控制临界值或准确度等级）。*解析：严格来说应为控制临界值，但在本题语境下，是对估算准确度等级的要求。*
