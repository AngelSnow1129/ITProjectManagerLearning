## 12.1 管理基础

### 12.1.1 质量与项目质量

#### 1. 质量的定义
*   **国际标准**：反映实体满足主体明确和隐含需求的能力的特性总和。
*   **国家标准**：一组固有特性满足要求的程度。

#### 2. 质量与等级 (重点)
*   **质量 (Quality)**：作为实现的性能或成果，是“好与坏”的区别。
*   **等级 (Grade)**：对用途相同但技术特性不同的可交付成果的级别分类。
*   **核心观点**：
    *   **质量低劣**（如有许多缺陷、用户手册杂乱）是个问题，**不可接受**。
    *   **等级低**（如功能有限、但无缺陷）**不是**问题，可以被接受。

#### 3. 预防与检查
*   **预防 (Prevention)**：保证过程中不出现错误（将质量设计到产品中）。
*   **检查 (Inspection)**：保证错误不落到客户手中（在产品完成后发现问题）。
*   **原则**：**预防胜于检查**。预防成本通常远低于检查或纠正错误的成本。

#### 4. 统计术语
*   **公差 (Tolerance)**：结果的可接受范围。
*   **控制界限 (Control Limits)**：在统计意义上稳定的过程或过程绩效的普通偏差的边界（通常为 $\pm3\sigma$）。

### 12.1.2 质量管理水平 (按有效性递增排序)
1.  **让客户发现缺陷**：代价最大（召回、商誉受损）。
2.  **控制质量 (QC)**：先检测和纠正，再发给客户（涉及评估成本和内部失败成本）。
3.  **质量保证 (QA)**：检查并纠正过程本身。
4.  **设计质量**：将质量融入规划和设计中。
5.  **质量文化**：在全组织创建关注质量的文化（最高境界）。

![](./images/第12章_项目质量管理/pic_004.jpg)

### 12.1.3 质量管理标准体系
1.  **GB/T 19000系列标准**。
2.  **全面质量管理 (TQM)**：全员、全过程、全组织。特征：全员参加、全过程、全面方法、全面结果。

### 12.1.4 管理新实践
*   **客户满意**：符合要求 + 适合使用。
*   **持续改进**：PDCA (计划-执行-检查-行动) 是基础。其他包括 TQM、六西格玛、精益六西格玛。
*   **管理层的责任**：质量问题的 85% 归咎于管理层，15% 归咎于员工。
*   **互利合作**：与供应商建立长期关系。

#### 六西格玛 (6$\sigma$) 数据对比
| 水平 | 缺陷率 (DPMO) | 影响示例 |
| :--- | :--- | :--- |
| **3$\sigma$** | 66,807 | 每年至少 54,000 次用药错误；每周电视中断 27 分钟。 |
| **6$\sigma$** | 3.4 | 25年才由一次用药错误；每周电视中断 2 秒。 |

## 12.3 规划质量管理

**作用**：为在整个项目期间如何管理和核实质量提供指南和方向。

![](./images/第12章_项目质量管理/pic_009.jpg)
![](./images/第12章_项目质量管理/pic_010.jpg)

### 12.3.1 输入
项目章程、项目管理计划、项目文件、EEF、OPA。
![](./images/第12章_项目质量管理/pic_011.jpg)

### 12.3.2 工具与技术

#### 1. 数据收集
*   **标杆对照 (Benchmarking)**：与可比项目（内部或外部）进行比较，识别最佳实践。
*   **头脑风暴**。
*   **访谈**。

#### 2. 数据分析
*   **成本效益分析**：比较可能的成本与预期的效益。
*   **质量成本 (COQ)**：

| 分类 | 子分类 | 描述 | 示例 |
| :--- | :--- | :--- | :--- |
| **一致性成本**<br>(规避失败) | **预防成本** | 预防特定产品质量低劣的成本 | 培训、文件过程、设备、完成时间 |
| | **评估成本** | 评估、测量、审计和测试的成本 | 测试、破坏性试验损失、检查 |
| **不一致成本**<br>(失败成本) | **内部失败成本** | 项目内部发现的失败 | 返工、报废 |
| | **外部失败成本** | 客户发现的失败（代价最高） | 债务、保修、业务流失 |

![](./images/第12章_项目质量管理/pic_012.jpg)

#### 3. 数据表现
*   **流程图 (Flowcharts)**：也称过程图，显示步骤顺序和分支。**SIPOC模型**（供应商-输入-过程-输出-客户）是常用形式。有助于估算质量成本，识别非增值环节。
    ![](./images/第12章_项目质量管理/pic_013.jpg)
    ![](./images/第12章_项目质量管理/pic_014.jpg)
*   **逻辑数据模型**：可视化组织数据，识别数据完整性问题。
*   **矩阵图**：展示因素、原因和目标之间的强弱关系（如L型、T型矩阵）。
*   **思维导图**：可视化组织信息。

#### 4. 其他
*   **测试与检查的规划**：决定如何测试（如 $\alpha$ 测试、$\beta$ 测试）。

### 12.3.3 输出
1.  **质量管理计划**：描述如何实施质量政策。包含质量标准、目标、角色职责、工具等。
    ![](./images/第12章_项目质量管理/pic_015.jpg)
2.  **质量测量指标**：具体的度量标准（如缺陷率、测试覆盖率、故障密度、可用性等）。
    ![](./images/第12章_项目质量管理/pic_016.jpg)

## 12.4 管理质量 (Quality Assurance)

**定义**：将质量管理计划转化为可执行的质量活动。
**核心作用**：
1.  提高实现质量目标的可能性。
2.  识别**无效过程**和导致质量低劣的原因。
3.  属于**过程关注**，不仅仅是产品。

*注意：在敏捷项目中，质量管理由所有团队成员执行；传统项目中通常由特定团队成员负责。*

![](./images/第12章_项目质量管理/pic_017.jpg)
![](./images/第12章_项目质量管理/pic_018.jpg)

### 12.4.1 输入
项目管理计划、项目文件（质量控制测量结果、风险报告）、OPA。
![](./images/第12章_项目质量管理/pic_019.jpg)

### 12.4.2 工具与技术

#### 1. 数据分析
*   **过程分析**：识别过程改进机会，检查非增值活动。
*   **根本原因分析 (RCA)**：识别问题根本原因并解决。

#### 2. 数据表现 (QC七工具中的部分)
*   **亲和图**：对潜在缺陷成因进行分类。
*   **因果图 (鱼骨图/石川图)**：识别问题的主要原因或根本原因。
*   **流程图**：展示引发缺陷的步骤。
*   **直方图**：展示缺陷数量、成因排列（如帕累托图）。
*   **矩阵图**：展示关系强弱。
*   **散点图**：展示两个变量之间的关系。

![](./images/第12章_项目质量管理/pic_020.jpg)
![](./images/第12章_项目质量管理/pic_021.jpg)
![](./images/第12章_项目质量管理/pic_022.jpg)
![](./images/第12章_项目质量管理/pic_023.jpg)
![](./images/第12章_项目质量管理/pic_024.jpg)
![](./images/第12章_项目质量管理/pic_025.jpg)

#### 3. 审计 (Audit) (重点)
*   **定义**：确定项目活动是否遵循了组织和项目的政策、过程与程序。
*   **执行者**：通常由外部团队（PMO、内部/外部审计师）执行。
*   **目标**：
    1.  识别最佳实践。
    2.  识别违规做法、差距及不足。
    3.  分享良好实践。
    4.  协助改进过程。
    5.  积累经验教训。

#### 4. 面向X的设计 (DfX)
旨在优化设计的特定方面（如可靠性、成本、安全性）。可降低成本、改进质量。

#### 5. 问题解决
定义问题 -> 识别根源 -> 生成方案 -> 选择方案 -> 执行 -> 验证。

#### 6. 质量改进方法
PDCA、六西格玛。

![](./images/第12章_项目质量管理/pic_026.jpg)

### 12.4.3 输出
1.  **质量报告**：包含质量管理问题、改善建议、控制质量发现的情况。
2.  **测试与评估文件**。
3.  变更请求。

![](./images/第12章_项目质量管理/pic_027.jpg)

## 12.5 控制质量 (Quality Control)

**定义**：核实项目可交付成果和工作已经达到主要干系人的质量要求，可供最终验收。
**核心**：关注**结果**（可交付成果），确保其完整、正确。

![](./images/第12章_项目质量管理/pic_028.jpg)

### 12.5.1 输入
项目管理计划、项目文件（测试与评估文件）、**可交付成果**、工作绩效数据。
![](./images/第12章_项目质量管理/pic_029.jpg)
![](./images/第12章_项目质量管理/pic_030.jpg)

### 12.5.2 工具与技术

#### 1. 数据收集
*   **核对单 (Checklists)**：结构化工具，核实步骤是否执行。
*   **核查表 (Check Sheets)**：计数表，收集属性数据（如缺陷计数）。
    ![](./images/第12章_项目质量管理/pic_031.jpg)
*   **统计抽样**：从总体中选取部分样本。
*   **问卷调查**。

#### 2. 检查 (Inspection)
检验工作产品，确定是否符合标准。也称审查、同行审查、审计或巡检。

#### 3. 测试/产品评估
包括单元测试、集成测试、黑盒/白盒测试等。

#### 4. 数据表现
*   **控制图 (Control Charts)** (重点)：
    *   用途：确定过程是否稳定，或是否具有可预测的绩效。
    *   **七点规则**（法则）：如果连续7个点落在均值上方或下方，或者连续7个点呈上升或下降趋势，则过程失控。
    *   **规格界限** vs **控制界限**：规格是客户要求的，控制界限是统计计算的。

![](./images/第12章_项目质量管理/pic_032.jpg)

### 12.5.3 输出
1.  **质量控制测量结果**：书面记录。
2.  **核实的可交付成果**：作为“确认范围”过程的输入，以便正式验收。
3.  工作绩效信息。
4.  变更请求。

![](./images/第12章_项目质量管理/pic_033.jpg)
![](./images/第12章_项目质量管理/pic_034.jpg)

(1) 以下质量管理水平有效性最高的是 ( )。

**【答案】A**

A. 规划质量管理
B. 管理质量
C. 控制质量
D. 改进质量
**【答案】C**

![](./images/第12章_项目质量管理/pic_035.jpg)

(3) 项目有关的质量成本中，属于不一致成本的是 ( )。
A. 预防成本
B. 评估成本
C. 破坏性试验损失成本
D. 外部失败成本
**【答案】D** (A、B、C属于一致性成本)

(4) 数据表现技术中，“可以对潜在缺陷成因进行分类，展示最应关注领域”的技术称为（ ）。
A. 因果图
B. 亲和图
C. 直方图
D. 散点图
**【答案】B**

(5) ( ) 不属于“控制质量”过程的输出。
A. 质量控制测量结果
B. 质量报告
C. 工作绩效信息
D. 变更请求
**【答案】B** (质量报告是“管理质量”的输出)

![](./images/第12章_项目质量管理/pic_036.jpg)

**判断题**
(1) 工作质量是指产品的使用价值及其属性，而产品质量则是工作质量的保证。 (×)
*(解析：反了。工作质量是产品质量的保证。)*
(2) 为引导变更，敏捷方法要求多个质量与审核步骤贯穿整个项目，而不是在面临项目结束时才执行。 (√)
(3) 对管理质量负有责任的岗位与团队，包括项目经理、项目团队、项目发起人、执行组织的管理层，甚至是客户。 (√)
(4) 在问卷调查中识别的缺陷相关成本可被视为COQ模型中的内部失败成本，给组织带来的影响会超出成本本身。 (×)
*(解析：问卷调查通常是针对客户的，识别出的缺陷属于外部失败成本。)*

关于控制质量的描述，不正确的是（ ）。
A. 控制质量的目的是在用户验收和最终交付之前，测量产品或服务的完整性、合规性和适用性
B. 控制质量时，控制图可用于确定一个过程是否稳定
C. 在瀑布模型或预测型项目中，控制质量活动通常由特定团队成员在整个项目生命周期中持续执行
D. 质量检查既可以针对单个活动的成果，也可以针对项目的最终产品
**【答案】C**
*(解析：在瀑布模型中，控制质量通常在特定时间点或阶段结束时执行，而非全生命周期持续执行。)*
