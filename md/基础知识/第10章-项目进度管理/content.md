## 10.1 管理基础

### 10.1.1 进度计划定义与要求
项目进度计划是沟通和管理干系人期望的工具。
**编制步骤**：选择方法（如关键路径法）$\rightarrow$ 输入特定数据（活动、日期、资源、依赖关系）$\rightarrow$ 创建进度模型 $\rightarrow$ 生成进度计划。

### 10.1.2 管理新实践
1.  **具有未完成项的迭代型进度计划**：适应型生命周期的滚动式规划。需求记录在**用户故事**中，按优先级排序开发。
2.  **按需进行的进度计划**：资源可用时立即提取任务（如看板系统）。适用于运营、增量研发或规模类似的项目。

**用户故事 (User Story)**：
*   **格式**：作为一个<角色>，我希望<功能>，以便<价值>。
*   **特征 (INVEST)**：独立的、可讨论的、有价值的、可估计的、小的 (Small)、可测试的。

## 10.5 排列活动顺序

**作用**：定义工作逻辑顺序。每项活动（除首尾）都至少有一项紧前和紧后活动。

![](./images/第10章_项目进度管理/pic_016.jpg)
![](./images/第10章_项目进度管理/pic_017.jpg)

### 10.5.1 输入
项目管理计划、项目文件、EEF、OPA。

### 10.5.2 工具与技术 (重点)

#### 1. 紧前关系绘图法 (PDM) / 单代号网络图 (AON)
*   **节点**代表活动，**箭线**代表逻辑关系。
*   **四种依赖关系**：
    *   **FS (完成到开始)**：最常用。前一个做完，后一个才能开始。
    *   **FF (完成到完成)**：前一个做完，后一个才能做完。
    *   **SS (开始到开始)**：前一个开始，后一个才能开始。
    *   **SF (开始到完成)**：很少用。前一个开始，后一个才能做完。

**节点表示法**：
*   **ES (最早开始)**, **EF (最早完成)**
*   **LS (最迟开始)**, **LF (最迟完成)**
*   公式：$EF = ES + \text{工期}$；$LS = LF - \text{工期}$

![](./images/第10章_项目进度管理/pic_018.jpg)
![](./images/第10章_项目进度管理/pic_020.jpg)
![](./images/第10章_项目进度管理/pic_021.jpg)

#### 2. 箭线图法 (ADM) / 双代号网络图 (AOA)
*   **箭线**代表活动，**节点**代表事件。
*   **虚活动 (Dummy Activity)**：用虚箭线表示，**不消耗时间也不消耗资源**，仅用于表达逻辑依赖关系。
*   **原则**：唯一代号；节点代号递增；同一节点流入/流出有共同紧前/紧后活动。

![](./images/第10章_项目进度管理/pic_023.jpg)
![](./images/第10章_项目进度管理/pic_025.jpg)

#### 3. 依赖关系类型
*   **强制性依赖 (硬逻辑)**：法律、合同或工作内在性质决定（客观限制，不可违背）。
    *   *例：地基建成后才能建地面结构。*
*   **选择性依赖 (软逻辑)**：基于最佳实践或特殊性质（可自由选择，快速跟进时需审查）。
    *   *例：先做管道再做电气（可并行，但按顺序风险低）。*
*   **外部依赖**：项目活动与非项目活动之间（团队不可控）。
    *   *例：等待政府听证会、外部硬件到货。*
*   **内部依赖**：项目活动之间（团队可控）。
    *   *例：机器组装完才能测试。*

![](./images/第10章_项目进度管理/pic_027.jpg)

#### 4. 提前量和滞后量
*   **提前量 (Lead)**：紧后活动提前开始的时间（负值）。
*   **滞后量 (Lag)**：紧后活动推迟开始的时间（正值）。

![](./images/第10章_项目进度管理/pic_028.jpg)

### 10.5.3 输出
1.  **项目进度网络图**：展示逻辑关系的图形。
2.  项目文件更新。

## 10.6 估算活动持续时间

**作用**：确定完成每个活动所需的时间量。
**考虑因素**：收益递减规律、资源数量、技术进步、员工激励（拖延症、帕金森定律）。

![](./images/第10章_项目进度管理/pic_031.jpg)
![](./images/第10章_项目进度管理/pic_032.jpg)

### 10.6.1 输入
项目管理计划、项目文件、EEF、OPA。

### 10.6.2 工具与技术

1.  **类比估算**：使用相似历史数据。成本低、快，但准确性低（粗略估算）。
2.  **参数估算**：利用统计关系（如：平米数 $\times$ 每平米工时）。准确性取决于参数模型的成熟度和数据可靠性。
3.  **三点估算** (PERT)：考虑不确定性。
    *   $T_O$ (乐观), $T_M$ (最可能), $T_P$ (悲观)。
    *   **三角分布**：$T_E = (T_O + T_M + T_P) / 3$
    *   **贝塔分布**：$T_E = (T_O + 4T_M + T_P) / 6$
4.  **自下而上估算**：逐层汇总，最准确。
5.  **数据分析**：
    *   **备选方案分析**。
    *   **储备分析**：
        *   **应急储备**：“已知-未知”风险。包含在**进度基准**中。PM可控。
        *   **管理储备**：“未知-未知”风险。**不包含**在进度基准中（属项目总持续时间）。PM不可控，需高层批准。

![](./images/第10章_项目进度管理/pic_033.jpg)
![](./images/第10章_项目进度管理/pic_034.jpg)
![](./images/第10章_项目进度管理/pic_035.jpg)

### 10.6.3 输出
1.  **持续时间估算**：不包括滞后量，可指出变动区间（如：2周 $\pm$ 2天）。
2.  **估算依据**。
3.  项目文件更新。

## 10.7 制定进度计划

**作用**：创建进度模型，落实执行和监控。是一个反复进行的过程。

![](./images/第10章_项目进度管理/pic_039.jpg)
![](./images/第10章_项目进度管理/pic_040.jpg)

### 10.7.1 输入
项目管理计划、项目文件、协议、EEF、OPA。

### 10.7.2 工具与技术 (核心考点)

#### 1. 关键路径法 (CPM)
*   **关键路径**：从起点到终点持续时间最长的路径。决定项目的最短工期。可能有多条。
*   **总浮动时间 (Total Float)**：不延误**项目完成日期**前提下，活动可以推迟的时间。
    *   $TF = LS - ES$ 或 $LF - EF$。
*   **自由浮动时间 (Free Float)**：不延误**紧后活动最早开始**前提下，活动可以推迟的时间。
    *   $FF = \min(\text{紧后活动 } ES) - \text{本活动 } EF$。

![](./images/第10章_项目进度管理/pic_042.jpg)
![](./images/第10章_项目进度管理/pic_044.jpg)

#### 2. 资源优化
*   **资源平衡 (Leveling)**：解决资源冲突或过度分配。往往导致**关键路径改变**（工期延长）。
*   **资源平滑 (Smoothing)**：利用浮动时间调整，**不改变关键路径**（完工日期不延误）。无法实现所有资源优化。
    *(口诀：平衡改工期，平滑不改)*

![](./images/第10章_项目进度管理/pic_045.jpg)
![](./images/第10章_项目进度管理/pic_046.jpg)
![](./images/第10章_项目进度管理/pic_047.jpg)
![](./images/第10章_项目进度管理/pic_048.jpg)

#### 3. 数据分析
*   **假设情景分析**：评估各种情景（如“如果...会怎样”）。
*   **模拟**：蒙特卡洛分析，计算概率分布。

#### 4. 进度压缩 (不缩减范围前提下)
*   **赶工 (Crashing)**：增加资源（如加班、加人），以最小成本换取时间。只适用于关键路径活动。可能增加**成本/风险**。
*   **快速跟进 (Fast Tracking)**：并行开展活动。可能导致**返工、风险增加**。

![](./images/第10章_项目进度管理/pic_050.jpg)
![](./images/第10章_项目进度管理/pic_051.jpg)

#### 5. 敏捷发布规划
基于路线图，提供高度概括的发布进度（3-6个月），确定迭代次数。

### 10.7.3 输出
1.  **进度基准**：经批准的进度计划。
2.  **项目进度计划**：
    *   **横道图 (甘特图)**：展示活动起止时间，易读，常用。
    *   **里程碑图**：仅标示主要可交付成果计划日期。
    *   **项目进度网络图**：展示逻辑关系。
3.  **进度数据**。
4.  **项目日历**：规定可用工作日/班次。

![](./images/第10章_项目进度管理/pic_053.jpg)
![](./images/第10章_项目进度管理/pic_054.jpg)

### 10.8.2 工具与技术

2.  **迭代燃尽图**：追踪迭代未完项，分析偏差趋势。

4.  **趋势分析**。
5.  **偏差分析**。

### 10.8.3 输出
1.  **工作绩效信息** (SV, SPI)。
2.  **进度预测**。
3.  变更请求。

**缩短工期方法总结 (补充背诵)**：
1.  **赶工** (增加资源/时间)。
2.  **快速跟进** (并行)。
3.  使用高素质资源。
4.  减小活动范围/降低要求 (需甲方同意)。
5.  改进方法/技术。
6.  加强质量管理 (减少返工)。

(9) 某项活动最可能时间8天，最乐观时间7天，最悲观时间12天，则该活动的期望持续时间为 ( )。
A. 7天
B. 8天
C. 9天
D. 10天
**【答案】C** ( $(7 + 4\times8 + 12) / 6 = 51 / 6 = 8.5$，取整或最接近选项，通常贝塔分布为 $(O+4M+P)/6$，此处若无贝塔说明默认也是贝塔，计算得8.5，若按三角分布 $(7+8+12)/3=9$。考题中通常优选贝塔，若无精确值选最接近。此处C为9，可能是按三角分布计算，或题目暗示取整。) *修正解析：按贝塔分布计算为8.5，按三角分布为9。若无特别说明，默认贝塔分布。但在本题选项中没有8.5，只有8和9。按照PMP考试惯例，若贝塔结果无选项，可尝试三角分布。这里选C(9天)符合三角分布结果。*

关于估算活动持续时间过程的工具与技术的描述，不正确的是（ ）。
A. 类比估算是一种精确的估算方法，适用于项目详细信息充分、项目需求明确的情况
B. 专家判断依赖于专家的可用性和经验，可能受到主观判断的影响
