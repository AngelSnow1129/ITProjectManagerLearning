## 18.1 干系人绩效域

干系人绩效域涉及与干系人相关的活动和职能。有效执行本绩效域可以实现的**预期目标**主要包含：
1.  与干系人建立高效的工作关系。
2.  干系人认同项目目标。
3.  支持项目的干系人提高了满意度，并从中收益。
4.  反对项目的干系人没有对项目产生负面影响。

### 18.1.1 绩效要点

为了有效地让干系人参与，项目经理可带领项目团队按以下步骤开展工作：

1.  **识别**：
    *   从高层级到详细逐层识别。
    *   注意识别那些难以发现的干系人。
    *   是一个持续的过程，需重复进行。
2.  **理解和分析**：
    *   了解干系人的感受、情绪、信念和价值观。
    *   **分析因素**：权力、作用、态度、信念、期望、影响程度、邻近性、利益等。
    *   需要对干系人分析工作进行保密。
    *   需要分析干系人之间的互动关系（联盟）。
3.  **优先级排序**：
    *   将管理焦点聚焦于权力和利益大的干系人。
    *   环境变化时需重新排序。
4.  **参与**：
    *   促进干系人参与项目，启发需求、谈判、解决问题和决策。
    *   运用软技能（积极倾听、冲突管理）和领导技能（创建愿景、批判性思维）。
    *   结合沟通管理方法。
    
5.  **监督**：
    *   监督干系人参与的数量和有效性。
    *   评估参与策略是否有效。
    *   通过**干系人满意度指标**评估（对话、审查会、问卷调查）。

### 18.1.2 与其他绩效域的相互作用
干系人参与项目的很多方面：
1.  为项目团队定义需求和范围，并进行优先级排序。
2.  参与并制定规划。
3.  确定可交付物和成果的验收及质量标准。
4.  关注绩效测量（如客户、高层管理人员）。



### 18.1.3 执行效果检查

| 预期目标 | 指标及检查方法 |
| :--- | :--- |
| **建立高效的工作关系** | **干系人参与的连续性**：通过观察、记录方式衡量。 |
| **干系人认同项目目标** | **变更的频率**：大量变更可能表明干系人未参与或与目标不一致。 |
| **支持者满意，反对者无负面影响** | **干系人行为**：表明是否支持或反对。<br>**干系人满意度**：调研、访谈。<br>**问题和风险**：审查问题日志和风险登记册。 |

## 18.2 团队绩效域

团队绩效域涉及项目团队人员有关的活动和职能。
**预期目标**：
1.  共享责任。
2.  建立高绩效团队。
3.  所有团队成员都展现出相应的领导力和人际关系技能。

### 18.2.1 绩效要点

#### 1. 项目团队文化
项目经理可以通过以下方法维护安全、尊重、无偏见的团队文化：
*   **透明**：有助于识别和分享。
*   **诚信**：职业道德和诚实。
*   **尊重**：尊重每个人及其贡献。
*   **积极的讨论**：处理不同见解，消除误解。
*   **支持**：解决问题，消除障碍。
*   **勇气**：勇于提出建议或异议。
*   **庆祝成功**：实时认可，激励团队。

#### 2. 高绩效项目团队
特征包括：开诚布公的沟通、共识、共享责任、信任、协作、适应性、韧性、赋能、认可。

#### 3. 领导力技能
*   **建立和维护愿景**：总结项目目的，作为激励工具。
*   **批判性思维**：理性、逻辑、基于证据的思维（元认知）。
*   **激励**：了解激励因素并开展合作。
*   **人际关系技能**：
    *   **情商 (EQ)**：
        1.  **自我意识**：了解自己的情绪、优势和劣势。
        2.  **自我管理**：控制破坏性感受和冲动。
        3.  **社交意识**：同理心，读懂非语言暗示。
        4.  **社交技能**：管理团队，建立关系。
        
        ![](images/pic_007.jpg)
    *   **决策**：
        *   **单方面决策**：快，但易出错，降低积极性。
        *   **群体决策**：包容性强，增加承诺，利用广泛知识；但耗时。
        *   **发散/汇聚模式**：先分别参与（发散），再汇聚确定首选方案。
    *   **冲突管理**：尊重沟通、聚焦问题、聚焦当前未来、共同寻找方案。



### 18.2.2 与其他绩效域的相互作用
团队技能融入项目的各个方面（规划、沟通、工作执行、测量等）。


### 18.2.3 执行效果检查

| 预期目标 | 指标及检查方法 |
| :--- | :--- |
| **共享责任** | **目标和责任心**：了解愿景，对成果承担责任。 |
| **建立高绩效团队** | **信任与协作程度**。<br>**适应变化的能力**（韧性）。<br>**彼此赋能**。 |
| **展现领导力和人际关系技能** | **管理和领导力风格适宜性**。 |

## 18.3 开发方法和生命周期绩效域

**预期目标**：
1.  开发方法与项目可交付物相符合。
2.  将项目交付与干系人价值紧密关联。
3.  项目生命周期由促进交付节奏的项目阶段和产生项目交付物所需的开发方法组成。

### 18.3.1 绩效要点

#### 1. 交付节奏
*   **一次性交付**：项目结束时交付（如流程再造）。
*   **多次交付**：不同时间交付多个组件。
*   **定期交付**：按固定计划（如每两周）。
*   **持续交付**：增量交付，小批量，自动化（适合数字化产品）。

#### 2. 开发方法
![](images/pic_011.jpg)


*   **预测型方法 (瀑布型)**：
    *   范围、进度、成本早期明确定义。
    *   适合：重大投资、高风险、建筑项目。
    
*   **混合型方法**：
    *   预测型与适应型的结合。
    *   适合：需求存在不确定性或风险，交付物可模块化。
    *   通常使用**迭代型**（澄清需求，最后一次迭代完成全部功能）或**增量型**（分批增加功能）方法。
    
*   **适应型方法 (敏捷)**：
    *   需求高度不确定和易变。
    *   迭代周期短，根据反馈不断演变。

#### 3. 开发方法的选择
影响因素：
*   **产品/服务/成果**：创新程度、需求确定性、范围稳定性、变更难易程度、交付物性质、风险、安全需求、法规。
*   **项目**：干系人参与度、进度制约、资金可用性。
*   **组织**：组织结构、文化、组织能力、团队规模和位置（敏捷适合7±2人，同地办公）。

### 18.3.2 与其他绩效域的相互作用
与干系人、规划、不确定性、交付、项目工作、团队绩效域密切相关。


### 18.3.3 执行效果检查

| 预期目标 | 指标及检查方法 |
| :--- | :--- |
| **开发方法与可交付物相符** | **产品质量和变更成本**：适宜的方法导致变更成本相对较小。 |
| **与干系人价值紧密联系** | **价值导向型项目阶段**：包含适当的退出标准。 |
| **生命周期结构适宜** | **适宜的交付节奏和开发方法**。 |

## 18.4 规划绩效域

**预期目标**：
1.  项目以有条理、协调一致的方式推进。
2.  应用系统的方法交付项目成果。
3.  对演变情况进行详细说明。
4.  规划投入的时间成本是适当的。
5.  规划内容充分。
6.  可根据变化进行调整。

### 18.4.1 绩效要点

#### 1. 规划的影响因素
*   **开发方法**：预测型（预先规划）、适应型（迭代规划）。
*   **项目可交付物**：建筑（大量前期规划）vs 产品开发（持续规划）。
*   **组织需求**。
*   **市场条件**。
*   **法律或法规限制**。

#### 2. 项目估算
影响估算的4个因素：
1.  **区间**：早期 $-25\% \sim +75\%$；进展良好时 $-5\% \sim +10\%$。
    ![](images/pic_020.jpg)
2.  **准确度**：估算的正确性。
3.  **精确度**：精准度（如“2天”比“本周”精确）。
    ![](images/pic_021.jpg)
4.  **信心**：随经验增长而增加。

**估算方法**：
*   **确定性估算**（点估算）与**概率估算**（区间+概率，如模拟）。
*   **绝对估算**与**相对估算**（如故事点）。
*   **基于工作流的估算**（周期和产量）。
*   **对不确定性的调整估算**（增加应急储备）。

#### 3. 其他规划内容
*   **团队组成和结构**：技能组合、地点、成本收益。
*   **沟通规划**：与干系人绩效域关联。
*   **实物资源规划**：物流、存储、处置。
*   **采购规划**：自制或外购分析。
*   **变更规划**：变更管理流程。
*   **度量指标和一致性**：确定度量指标（只测量重要的东西），保证规划与实际一致。

### 18.4.2 与其他绩效域的相互作用
规划贯穿整个生命周期，与不确定性、交付、工作绩效域整合。


### 18.4.3 执行效果检查

| 预期目标 | 指标及检查方法 |
| :--- | :--- |
| **有条理、协调一致** | **绩效偏差**：处于临界值范围内。 |
| **应用系统方法** | **规划的整体性**：无差距或不一致。 |
| **详细说明演变** | **规划的详尽程度**：信息适当。 |
| **时间成本适当** | **规划适宜性**。 |
| **内容充分** | **规划的充分性**。 |
| **可调整** | **可适应变化**：变更控制过程有效。 |

## 18.5 项目工作绩效域

**预期目标**：
1.  高效且有效的项目绩效。
2.  适合项目和环境的项目过程。
3.  干系人适当的沟通和参与。
4.  对实物资源、采购进行了有效管理。
5.  有效处理了变更。
6.  通过持续学习和过程改进提高了团队能力。

### 18.5.1 绩效要点
1.  **项目过程**：定期审查过程效率和效果。优化方法包括：
    *   **精益生产法**（价值流图，消除浪费）。
    *   **回顾会议**（经验教训）。
    *   **价值导向审查**。
2.  **项目制约因素**：平衡变化中的制约因素（如日期、法规、预算）。
3.  **专注于工作过程和能力**：交付价值 + 保护团队能力。
4.  **管理沟通和参与**。
5.  **管理实物资源**：减少搬运、消除等待、最小化浪费。
6.  **处理采购事宜**。
7.  **监督新工作和变更**：
    *   敏捷：待办事项列表优先级排序。
    *   预测型：变更控制委员会 (CCB)。
8.  **学习和持续改进**：知识转移，丰富组织过程资产。



### 18.5.2 与其他绩效域的相互作用
促进规划、交付、度量，支持团队互动和干系人参与。


### 18.5.3 执行效果检查
通过状态报告、过程审计、资源利用率、采购审计、变更日志、团队绩效报告等检查。

## 18.6 交付绩效域

**预期目标**：
1.  项目有助于实现业务目标和战略。
2.  实现了预期成果。
3.  在预定时间内实现了项目收益。
4.  团队对需求有清晰理解。
5.  干系人接受并满意。

### 18.6.1 绩效要点
1.  **价值的交付**：可分批交付或项目结束时交付。
2.  **可交付物**：
    *   **需求启发**：清晰、可核实、一致、完整。
    *   **不断演变的需求**：原型、演示。
    *   **管理需求**：需求跟踪矩阵。
    *   **定义范围和管理变更**。
3.  **质量**：聚焦于绩效水平，平衡质量成本。

### 18.6.2 与其他绩效域的相互作用
是规划工作的终点，基于开发方法，由项目工作促使。


### 18.6.3 执行效果检查
检查目标一致性、项目完成度、项目收益、需求稳定性、干系人满意度。

## 18.7 度量绩效域

**预期目标**：
1.  对项目状况充分理解。
2.  数据充分，支持决策。
3.  及时采取行动。
4.  基于预测和评估作出决策。

### 18.7.1 绩效要点

#### 1. 制定有效的度量指标
*   **关键绩效指标 (KPI)**：
    *   **提前指标 (Leading)**：预测未来趋势（如风险管理缺失）。
    *   **滞后指标 (Lagging)**：反映过去绩效（如进度偏差、已完成数量）。
*   **有效指标特征 (SMART)**：具体的(S)、有意义的(M)、可实现的(A)、具有相关性(R)、具有及时性(T)。

#### 2. 度量内容及相应指标
*   **可交付物**：缺陷信息、技术绩效。
*   **交付**（适应型项目常用）：
    *   **在制品 (WIP)**。
    *   **提前期** (Lead Time)：进入待办到完成的时间。
    *   **周期时间** (Cycle Time)：开始做到完成的时间。
    *   **队列大小** (利特尔法则)。
    *   **批量大小**。
    *   **过程效率** (增值时间/总时间)。
*   **基准绩效**：SV, SPI, CV, CPI。
*   **资源**：利用率、成本。
*   **价值**：成本效益比、ROI、NPV。
*   **干系人**：净推荐值 (NPS)、情绪图、士气、离职率。
    ![](images/pic_038.jpg)
*   **预测型**：ETC, EAC, TCPI, 回归分析, 产量分析。

#### 3. 展示度量信息和结果
*   **仪表盘**：信号灯图 (RAG)。
    ![](images/pic_040.jpg)
    ![](images/pic_041.jpg)
*   **大型可见图表 (BVC) / 信息发射源**：低科技高触感。
    ![](images/pic_042.jpg)
*   **任务板**：看板方式。
    ![](images/pic_044.jpg)
*   **燃烧图**：燃起图 (Burnup) / 燃尽图 (Burndown)。
    ![](images/pic_045.jpg)

#### 4. 度量陷阱
*   **霍桑效应**：度量本身会影响行为。
*   **虚荣指标**：对决策无帮助。
*   **士气低落**：目标无法实现。
*   **误用度量指标**。
*   **确认偏见**。
*   **相关性与因果关系混淆**。

#### 5. 基于度量进行诊断
制定临界值，制定诊断计划。

## 18.8 不确定性绩效域

**不确定性包含**：
*   **风险**：不可知未来事件。
*   **模糊性**：不了解当前或未来状况。
*   **复杂性**：动态系统，不可预测结果。

**预期目标**：
1.  了解运行环境。
2.  积极识别、分析和应对。
3.  了解依赖关系。
4.  预测威胁和机会。
5.  最小化负面影响，利用机会。
6.  有效利用储备。


### 18.8.1 绩效要点
1.  **风险**：确定风险临界值。
2.  **模糊性**：
    *   **概念模糊性**：缺乏理解（定义术语）。
    *   **情景模糊性**：多个结果（渐进明细、实验、原型法）。
3.  **复杂性**：
    *   **基于系统**：解耦、模拟。
    *   **重新构建**：多样性、平衡。
    *   **基于过程**：迭代、参与、故障保护。
4.  **应对方法**：收集信息、为多种结果做准备（备份/应急计划）、集合设计（权衡）、增加韧性。

**练习题：**

(1) 促进干系人参与的步骤包括识别、理解、（ ）、优先级排序、参与和（ ）。
A. 分析 变更
B. 分析 监督
C. 效果评价 监督
D. 效果评价 变更
**【参考答案】B**

(2) 有效执行团队绩效域可以实现的预期目标，不包括（ ）。
A. 共享责任
B. 建立高绩效团队
C. 所有团队成员都展现出相应的领导力和人际关系技能
