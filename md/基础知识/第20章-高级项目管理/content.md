## 20.1 项目集与项目组合基础

### 20.1.1 概念辨析

*   **项目 (Project)**：为创造独特的产品、服务或成果而进行的临时性工作。
*   **项目集 (Program)**：是一组相互关联且被协调管理的项目、子项目集和项目集活动，以便获得分别管理无法获得的利益。
    *   *示例*：建造一个包含电影院、美食广场、超市的购物广场。这三个场所的建设可视为独立项目，但统一作为项目集管理更有利于资源调度。
*   **项目组合 (Portfolio)**：是为了实现战略目标而组合在一起管理的项目、项目集、子项目组合和运营工作。
    *   *示例*：经营购物广场的任务。除了建设（项目集），还包括开业宣传（营销活动）、后期运营等。这些工作为了同一个战略目标（盈利）而整合在一起。

**三者关系总结：**
*   **项目组合**：服务于**战略目标**。
*   **项目集**：关注项目间的**关联关系**，实现更大效益。
*   **项目**：可以单独存在，也可以存在于项目组合和项目集中。

## 20.2 项目集管理标准

![](./images/第20章_高级项目管理/pic_007.jpg)

### 20.2.1 角色和职责

在项目集管理中，主要角色包括项目集发起人、项目集指导委员会、项目集经理等。

**1. 项目集发起人**
*   为项目集提供资金，确保项目集目标与战略愿景保持一致。
*   使效益实现交付。
*   消除项目集管理与交付的困难和障碍。

**2. 项目集指导委员会**
*   为项目集提供治理支持（监督、控制、整合和决策）。
*   提供有能力的治理资源，监督不确定性和复杂性。
*   确保项目集目标符合组织战略。
*   批准项目集建议、变更、收尾和终止。
*   解决上报的问题和风险。
*   审查预期效益和效益交付。

![](./images/第20章_高级项目管理/pic_009.jpg)

**3. 项目集经理**
*   在项目集管理绩效域内开展工作。
*   与项目经理、其他项目集经理、项目组合经理交互，确保资源和优先级。
*   与治理机构、发起人合作，确保与组织战略一致。
*   与运营负责人交互，确保获得运营支持并维持效益。
*   确保项目集组件的重要性被认可。
*   为项目集团队提供管理决策。

![](./images/第20章_高级项目管理/pic_010.jpg)

### 20.2.2 项目集管理绩效域

项目集管理绩效域包括：项目集战略一致性、项目集效益管理、项目集干系人参与、项目集治理、项目集生命周期管理。

![](./images/第20章_高级项目管理/pic_011.jpg)

**1. 项目集战略一致性**
确保项目集输出和成果与组织目标保持一致。
*   通过可行性研究和评估验证交付效益。
*   项目集章程用于衡量成功与否。
*   制订项目集路线图，与干系人沟通。

**2. 项目集效益管理**
主要活动包括：**效益识别、效益分析和规划、效益交付、效益移交、效益维持**。

**3. 项目集干系人参与**
识别和分析干系人需求、管理期望。
*   主要活动：识别、分析、参与规划、参与、沟通。

**4. 项目集治理**
实现和执行项目集决策，维护项目集监督。
*   重点是通过建立系统和方法，供发起组织定义、授权、监督和支持项目集。
*   通过对建议做出签署或批准的评审与决策活动来实现。

![](./images/第20章_高级项目管理/pic_012.jpg)

**5. 项目集生命周期管理**
分为三个主要阶段：
*   **项目集定义阶段**：构建和批准项目集，制定路线图、评估和章程，制订项目集管理计划。
*   **项目集交付阶段**：
    *   组件授权与规划。
    *   组件监督与整合。
    *   组件移交与收尾。
*   **项目集收尾阶段**：将效益移交给维护组织，正式结束或提前终止。

## 20.3 项目组合管理标准

### 20.3.1 角色和职责

**1. 项目组合经理**
负责建立和实施项目组合管理，扮演架构师、促成者、引导者及分析师角色。
*   向治理机构传达组件与战略的一致性。
*   依据战略获得影响和价值。
*   管理资源分配，协调组件实施。
*   向高级管理层汇报进展。

**2. 其他角色**
*   **发起人**：提供资源和支持，是资源分配和成功的捍卫者。
*   **项目组合治理机构**：指导监督，评估绩效，对投资和优先级做决策。
*   **PMO**：提供支持、集中管理和协调的组织实体。
*   **项目组合分析师**：识别、分析和追踪依赖关系，推荐改进方案。
*   **项目集经理**：确保项目集与项目组合管理计划一致。
*   **项目经理**：提供项目绩效指标。
*   **变更控制委员会 (CCB)**：审查变更请求。

![](./images/第20章_高级项目管理/pic_014.jpg)

### 20.3.2 项目组合管理绩效域

包括：项目组合生命周期、战略管理、治理、产能与能力管理、干系人参与、价值管理、风险管理。

**1. 项目组合生命周期**
由**启动、规划、执行、优化**四个阶段组成。

*   **启动阶段**：验证战略，识别组件，定义长期路线图（财务、绩效、沟通、治理等）。
*   **规划阶段**：制订并评审项目组合管理计划。
    *   活动包括：范围、预算、依赖关系、风险、资源、优先级排序、治理确认、衡量标准等。
    ![](./images/第20章_高级项目管理/pic_015.jpg)
*   **执行阶段**：通过组件和运营实施。
    *   活动包括：组件交付、管理风险、沟通汇报、重新排序、监督收益、资源管理。
*   **优化阶段**：最大化可用条件和资源，使其尽可能高效。

**2. 项目组合战略管理**
使组织的行动符合高级管理层和干系人的期望。

**3. 项目组合治理**
是一套实践、职能与过程。
*   **决策制定职能**：提供治理结构，授予管理权力。
*   **监管职能**：支持决策与导向。
*   **控制职能**：监控、测量和报告。
*   **整合职能**：支持战略一致性。

**4. 项目组合产能与能力管理**
以一系列**工具和实践**来识别、分配和优化资源。
*   **产能管理**涉及4个类别：人力资本、财务成本、资产、智力资本。

**5. 项目组合干系人参与**
迭代步骤：定义和识别、分析、规划参与、识别沟通方法、管理沟通。

**6. 项目组合价值管理**
关键活动：协商期望价值、最大化价值、实现价值、测量价值、报告价值。

**7. 项目组合风险管理**
关键要素：规划、识别、评估、应对。

## 20.4 组织级项目管理 (OPM)

**定义**：OPM 是通过整合项目组合、项目集和项目管理，连接其与组织驱动因素和组织流程来提升组织能力，从而实现战略目标。

![](./images/第20章_高级项目管理/pic_017.jpg)

### 20.4.1 业务价值与评估
*   OPM 旨在为组织主动创造价值。
*   **业务评估**是建立 OPM 框架的必要组件。需确定关键绩效指标，尽可能通过财务量化方式确定收益（ROI）。

### 20.4.2 OPM 框架要素
关键要素包括：**OPM治理、OPM方法论、知识管理、人才管理**。

1.  **OPM 方法论**：需要裁剪，以匹配特定需求。
2.  **知识管理**：侧重于绩效改进、创新、经验分享。
3.  **人才管理**：评估和提供个人绩效反馈。
4.  **OPM 治理**：
    *   *高管治理实体*：制定战略，优先级调整。
    *   *OPM 治理实体*：确保基础架构与战略一致。
    *   *项目组合/项目集治理实体*：报告收益和问题。
    *   *项目管理治理实体*：传递变更，识别影响。

![](./images/第20章_高级项目管理/pic_018.jpg)

### 20.4.3 OPM 成熟度模型
OPM 成熟度是指组织以可预测、可控制和可靠的方式交付期望战略成果的能力水平。

**成熟度级别：**
1.  **初始或临时的 OPM**：不稳定，依赖个人经验，流程无序。
2.  **项目层级采用 OPM**：在项目级有计划和控制，但未组织级统一，存在差异。
3.  **组织定义的 OPM**：标准流程已定义并裁剪，过程资产可复用，绩效可预测。
4.  **量化管理的 OPM**：决策数据驱动，实现稳定性监控和精细化管理。
5.  **持续优化的 OPM**：专注于持续改进，促进敏捷和创新。

## 20.5 量化项目管理

**定义**：以数据为基础，用统计或其他量化方法分析运行状态，对关键决策点进行监控的管理模式。

### 20.5.1 理论及应用

**1. 统计过程控制 (SPC)**
*   一种预防性方法，强调**全员参与**，重点在于**过程**。
*   过程波动性：**正常波动**（不可避免）、**异常波动**（特殊成因，需识别管理）。

**2. 六西格玛 (6σ)**
*   **DMAIC 模式**：定义 (Define)、度量 (Measure)、分析 (Analyze)、改进 (Improve)、控制 (Control)。
*   特征：基于数据统计分析，强调管理流程、指标的量化。

**3. CMMI 高成熟度特征**
*   量化的目标管理机制。
*   过程能力量化监控机制。
*   目标的量化预测能力。
*   基于量化的持续优化机制。

![](./images/第20章_高级项目管理/pic_021.jpg)

### 20.5.2 组织级量化管理

**前提**：组织已定义标准过程，收集的度量数据具备统计意义。
**内容**：定义目标、识别关键过程、建立度量体系、建立过程性能基线、建立过程性能模型。

**相关性分析 (Pearson 系数)**：
使用皮尔森 (Pearson) 相关系数 $r$ 判定因子 $x$ 与结果 $y$ 的相关性，范围 $[-1, 1]$。

![](./images/第20章_高级项目管理/pic_023.jpg)

*   **强相关**：$0.8 \le |r| \le 1$
*   **中度相关**：$0.5 \le |r| < 0.8$
*   **弱相关**：$0.3 \le |r| < 0.5$
*   **基本不相关**：$0 \le |r| < 0.3$

> 注意：因子间不可存在较高相关性，否则会导致多重共线问题。

### 20.5.3 项目级量化管理
1.  **项目过程性能目标定义**：使用蒙特卡洛模拟预测目标达成概率。
2.  **过程优化组合**：判定最优过程组合方案。
3.  **过程性能监控**：监控稳定性（控制图）和符合性。
4.  **项目性能预测**：在里程碑处预测最终目标达成性。

## 20.6 项目管理实践模型

### 20.6.1 CMMI 模型 (V2.0)

**四大能力域：**
1.  **行动 (Doing)**：生产和提供优秀解决方案。
2.  **管理 (Managing)**：策划和管理解决方案实施。
3.  **使能 (Enabling)**：支持解决方案实施和交付。
4.  **提高 (Improving)**：维持和提高效率效能。

**5个成熟度级别：**
*   **第1级 (初始级)**：基本执行，无完整实践，临时的。
*   **第2级 (管理级)**：遵守计划与流程，项目级管理，不依赖组织标准资产。
    ![](./images/第20章_高级项目管理/pic_024.jpg)
*   **第3级 (定义级)**：使用组织标准过程（可裁剪），过程资产复用。
*   **第4级 (量化管理级)**：管理量化、可预测，使用统计技术监控稳定性。
*   **第5级 (优化级)**：持续优化，预防问题，技术革新。
    ![](./images/第20章_高级项目管理/pic_025.jpg)

**改进步骤**：定义目标 -> 建立团队 -> 差距分析 -> 培训定义 -> 部署 -> 评估。

### 20.6.2 PRINCE2 模型

基于流程的项目管理方法。
*   **7大原则**：持续业务验证、吸取教训、定义角色职责、按阶段管理、例外管理、关注产品、根据项目剪裁。
*   **7大主题**：立项评估、组织、质量、计划、风险、变更、进展。
*   **7大流程**：准备、指导、启动、阶段控制、产品交付、阶段边界、收尾。
