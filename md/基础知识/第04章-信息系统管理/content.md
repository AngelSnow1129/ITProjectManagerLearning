## 4.1 管理方法

### 4.1.1 管理基础

**1. 层次结构**
信息系统包括四个要素：**人员、技术、流程、数据**。
管理层监督系统的设计和结构，监控整体性能，并制定业务需求和战略。

**2. 系统管理领域**
*   **规划和组织**：针对整体组织、战略和支持活动。
*   **设计和实施**：针对解决方案的定义、采购、实施及整合。
*   **运维和服务**：针对服务的运行交付和支持（含安全）。
*   **优化和持续改进**：针对性能监控及与目标的一致性管理。

### 4.1.2 规划和组织

**1. 规划模型：信息系统战略三角**
成功的组织需平衡**业务战略、信息系统、组织机制**之间的一致性。
![](./images/第4章_信息系统管理/pic_005.jpg)

**2. 组织模型（迈克尔·波特竞争力优势模型）**
*   **总成本领先战略**：成为市场上成本最低的生产者。
*   **差异性战略**：在市场上显得独特。
*   **专注化战略**：限制在狭窄的细分市场（专注成本或专注差异化）。
![](./images/第4章_信息系统管理/pic_006.jpg)

**3. 组织机制战略（莱维特钻石模型）**
关键组成部分：**人员、结构、任务、信息与控制**。

**4. 信息系统战略**
包括：硬件、软件、网络、数据。

### 4.1.3 设计和实施

**架构模式**

| 模式 | 描述 | 别称 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **集中式架构** | 大型中央计算机处理所有功能，通常在数据中心。 | 主机架构 | 需要易于管理，业务高度集中。 |
| **分布式架构** | 计算能力分散在多台设备（服务器、PC等）之间。 | 基于服务器的架构 | 担心可伸缩性，业务非集中化。 |
| **面向服务的架构 (SOA)** | 将大程序分解为互连的服务（SaaS, Web服务）。 | 基于Web的架构 | 需要敏捷架构，对新应用和快速迭代要求高。 |

![](./images/第4章_信息系统管理/pic_011.jpg)

### 4.1.4 运维和服务

**1. IT服务管理 (ITSM)**
*   **服务台 (Service Desk)**：用户体验感知窗口，沟通界面，官方接口。
*   **事件管理**：处理计划外中断或服务质量下降（故障）。
*   **问题管理**：寻找事件的根本原因。
*   **变更管理**：改变配置项状态。
*   **配置管理**：管理配置项信息及其关系。
*   **发布管理**：计划和实施变更。
*   **服务级别管理**：定义、记录服务级别并达成一致 (SLA)。
*   **容量管理**：确认容量满足需求。
*   **服务连续性管理**：确保持续提供服务的能力。
*   **可用性管理**：设计、监控、评价服务的可用性。

### 4.1.5 优化和持续改进

*   **PDCA循环**：Plan（计划）、Do（执行）、Check（检查）、Act（处理）。
*   **六西格玛 DMAIC/DMADV**：
    *   **DMAIC**：定义、度量、分析、改进、控制。
    *   **DMADV**：定义、度量、分析、设计、验证。

## 4.2 管理要点

### 4.2.1 数据管理

**DCMM (数据管理能力成熟度评估模型)**
定义了8个核心能力域：数据战略、数据治理、数据架构、数据应用、数据安全、数据质量、数据标准、数据生存周期。

**DCMM 成熟度等级：**
1.  **初始级**：被动式管理，无统一流程。
2.  **受管理级**：意识到数据是资产，制定了管理流程。
3.  **稳健级**：数据是重要资产，**标准化**管理流程。
4.  **量化管理级**：数据是竞争优势资源，效率可**量化**分析。
5.  **优化级**：数据是生存发展基础，流程实时优化，行业最佳实践。

![](./images/第4章_信息系统管理/pic_014.jpg)

### 4.2.2 运维管理

**1. IT运维能力模型 (GB/T 28827.1)**
包含四要素：
*   **人员**：选人做事。
*   **资源**：保障做事。
*   **技术**：**高效**做事。
*   **过程**：**正确**做事。

![](./images/第4章_信息系统管理/pic_017.jpg)

**2. 智能运维 (AIOps)**
*   **关键过程**：场景分析、场景构建、场景交付、效果评估。
*   **七大智能特征**：能感知、会描述、自学习、会诊断、可决策、自执行、自适应。

### 4.2.3 信息安全管理

**1. CIA三要素**
*   **保密性 (Confidentiality)**
*   **完整性 (Integrity)**
*   **可用性 (Availability)**

**2. 网络安全等级保护 (GB/T 22240)**
等级划分依据：受破坏后对客体（公民/法人权益、社会秩序/公共利益、国家安全）的侵害程度。

| 等级 | 公民、法人权益 | 社会秩序、公共利益 | 国家安全 |
| :--- | :--- | :--- | :--- |
| **第一级** | 损害 | - | - |
| **第二级** | 严重损害 | 危害 | - |
| **第三级** | - | **严重危害** | **危害** |
| **第四级** | - | **特别严重危害** | **严重危害** |
| **第五级** | - | - | **特别严重危害** |
