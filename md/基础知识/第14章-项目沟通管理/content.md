## 14.1 管理基础

### 14.1.1 沟通的定义与组成
项目沟通管理由两部分组成：
1.  **制定策略**：确保沟通对干系人行之有效。
2.  **执行活动**：落实沟通策略。

沟通是指用各种可能的方式来发送或接收信息。形式包括：书面、口头、正式或非正式、手势动作、媒体形式、遣词造句。

### 14.1.2 沟通模型

**关键要素：**
*   **编码**：把思想或想法转化为他人能理解的语言。
*   **信息和反馈信息**：编码过程所得到的结果。
*   **媒介**：用来传递信息的方法。
*   **噪声**：干扰信息传输和理解的一切因素。
*   **解码**：把信息还原成有意义的思想或想法（**最易发生误解的环节**）。

![](images/pic_003.jpg)


**沟通模型的5个基本状态：**
1.  **已发送**：信息已发送。
2.  **已收到**：对方已收到信息，但不代表有意图读取或理解。
3.  **已理解**：正确消化和理解内容。
4.  **已认可**：理解了信息并不代表同意观点。在两个有主见的人之间达成一致既复杂又耗时。
5.  **已转化为积极的行动**：这是最难的一环，通常需要反复沟通，在一定的监督或帮助下才能完成。

### 14.1.3 沟通分类

| 类型 | 说明 |
| :--- | :--- |
| **内部沟通** | 项目内部或组织内部的干系人。 |
| **外部沟通** | 外部干系人（客户、供应商、其他项目、组织、政府、公众）。 |
| **正式沟通** | 报告、正式会议、会议纪要、简报、演示。 |
| **非正式沟通** | 电子邮件、社交媒体、网站、非正式临时讨论。 |
| **层级沟通（垂直）** | 向上（高层）、向下（团队成员）。 |
| **层级沟通（水平）** | 横向（同级项目经理或其他人员）。 |
| **官方沟通** | 年报、呈交监管机构或政府部门的报告。 |
| **非官方沟通** | 灵活（往往为非正式）的手段。 |
| **书面沟通** | 社交媒体、网站、媒体发布、文件。 |
| **口头沟通** | 口头（词语和音调）及非口头（肢体语言）。 |



### 14.1.4 沟通技巧

**有效沟通的3个基本属性：**
1.  沟通目的明确。
2.  尽量了解沟通接收方，满足其需求及偏好。
3.  监督并衡量沟通的效果。

**书面沟通的5C原则：**
1.  **正确的语法和拼写 (Correctness)**。
2.  **简洁的表述 (Conciseness)**。
3.  **清晰的目的和表述 (Clarity)**。
4.  **连贯的思维逻辑 (Coherence)**。
5.  **善用控制语句和承接 (Controlling)**。

**项目经理需配合的沟通技巧：**
*   积极倾听。
*   理解文化和个人差异。
*   识别、设定并管理干系人期望。
*   强化技能。



### 14.1.5 管理新实践
1.  **将干系人纳入项目评审范围**：定期评审干系人及其态度变化。
2.  **让干系人参加项目会议**：甚至包括外部干系人。
3.  **社交工具的使用**：支持信息交换，建立信任和社群关系。
4.  **多面性沟通方法**：根据文化、背景选择合适的技术、语言、媒介。

## 14.3 规划沟通管理

需在项目生命周期**早期**制定，并**定期审查**（如干系人变化或新阶段开始时）。

![](images/pic_010.jpg)


### 14.3.1 输入
1.  **项目章程**
2.  **项目管理计划**
3.  **项目文件**（需求文件、干系人登记册）
4.  **事业环境因素**
5.  **组织过程资产**

### 14.3.2 工具与技术

#### 1. 沟通需求分析
确定干系人的信息需求（类型、格式、价值）。
*   **计算公式**：沟通渠道 $N \times (N-1) / 2$，其中 $N$ 为干系人数量。

#### 2. 沟通技术
影响选择的因素：紧迫性、可用性与可靠性、易用性、项目环境、敏感性和保密性。

#### 3. 沟通模型
从基本的线性模型到互动的、复杂的模型。

#### 4. 沟通方法 (重点)
*   **互动沟通**：两方或多方实时多向交换。效率最高，如会议、电话、视频。
*   **推式沟通**：向特定接收方发送。确保发送，**不确保**送达或理解。如邮件、报告、备忘录。
*   **拉式沟通**：适用于大量复杂信息或大量受众。接收方自行访问。如门户网站、知识库、在线课程。

**沟通需求实现方式：**
*   **人际沟通**：个人之间（面对面）。
*   **小组沟通**：3-6人。
*   **公众沟通**：单个演讲者面向一群人。
*   **大众传播**：发送者与大量目标受众（联系程度最低）。
*   **网络和社交工具沟通**：多对多。

#### 5. 人际关系与团队技能
*   **沟通风格评估**：识别偏好的方法、形式和内容。常用于不支持项目的干系人。
*   **政策意识**：对权力关系的认知，理解谁能施加影响。
*   **文化意识**：理解差异并调整策略。

#### 6. 数据表现
*   **干系人参与度评估矩阵**：显示当前与期望参与度的差距。

#### 7. 会议

### 14.3.3 输出

#### 1. 沟通管理计划
主要包括：
1.  干系人的沟通需求。
2.  需沟通的信息（语言、形式、内容、详细程度）。
3.  上报步骤。
4.  发布信息的原因。
5.  发布时限和频率（及确认收到）。
6.  负责沟通的人员。
7.  负责授权保密信息发布的人员。
8.  接收信息的人员或群体。
9.  传递信息的方法或技术。
10. 沟通资源（时间、预算）。
11. 更新与优化计划的方法。
12. 通用术语表。
13. 信息流向图、工作流程。
14. 制约因素（法规、技术等）。
15. 会议、邮件等的指南和模板。
16. 项目网站和软件管理。



#### 2. 项目管理计划（更新）
#### 3. 项目文件（更新）

## 14.4 管理沟通

作用：促成项目团队与干系人之间的有效信息流动。

![](images/pic_014.jpg)

### 14.4.1 输入
1.  **项目管理计划**
2.  **项目文件**：
    *   **变更日志**：传达变更及审批情况。
    *   **问题日志**：传达问题解决情况。
    *   **经验教训登记册**：改进沟通效率。
    *   **质量报告**：传达质量问题和改进建议。
    *   **风险报告**：传达整体风险和单个风险概述。
    *   **干系人登记册**。
3.  **工作绩效报告**：如状态报告、进展报告、挣值图表、燃尽图等。
4.  **事业环境因素**
5.  **组织过程资产**



### 14.4.2 工具与技术
1.  **沟通技术**
2.  **沟通方法**
3.  **沟通技能**：
    *   **沟通胜任力**：明确目的、建立关系、领导行为。
    *   **反馈**：指导、辅导和磋商。
    *   **非口头技能**：肢体语言、语调、面部表情、眼神交流（镜像模仿）。
    *   **演示**：正式交付信息和文档（报告进度、背景信息、提升形象等）。
4.  **项目管理信息系统 (PMIS)**
5.  **项目报告**：收集和发布项目信息（虽然WPR是监控的输出，但本过程编制临时报告、演示等）。
6.  **人际关系与团队技能**：
    *   **积极倾听**：告知已收到、澄清确认、消除障碍。
    *   **冲突管理**。
    *   **文化意识**。
    *   **会议管理**：确保高效。
    *   **人际交往**。
    *   **政策意识**。
7.  **会议**



### 14.4.3 输出
1.  **项目沟通记录**：绩效报告、状态、进度、成本、演示等。
2.  **项目管理计划（更新）**
3.  **项目文件（更新）**
4.  **组织过程资产（更新）**

## 本章练习与真题

**练习题**

(1) 沟通的基本模型用于显示信息如何在双方之间被发送和接收，日常与人交往过程中发生的误解，通常在 ( ) 环节发生。
A. 编码
B. 解码
C. 媒介
D. 信息
**【答案】B**

(2) 书面沟通的5C原则不包括 ( )。
A. 正确的语法和拼写
B. 连贯的思维逻辑
C. 详细的表述
D. 善用控制语句和承接
**【答案】C**
> 解析：5C原则包括简洁 (Concise)，而C选项是“详细”，这往往与简洁相悖。



(3) 规划沟通管理的主要作用不包括 ( )。
A. 及时向干系人提供相关信息
B. 引导干系人有效参与项目
C. 是编制书面沟通计划
D. 确保满足项目及其干系人的信息需求
**【答案】D**
> 解析：D选项“确保满足信息需求”是**监督沟通**的主要作用。

(4) 你正在组织项目沟通协调会，参加会议的人数为12人，沟通渠道有（ ）条。
A. 66
B. 72
C. 96
D. 132
**【答案】A**
> 解析：公式 $12 \times (12-1) / 2 = 66$。

(5) 适用于管理沟通过程的沟通技能不包括 ( )。
A. 反馈
B. 非口头技能
C. 演示
D. 社交媒体管理
**【答案】D**



**判断题**
(1) 解码是把思想或想法转化为他人能理解的语言。
(2) 规划沟通管理的输出包括干系人登记册。
(3) 监督沟通的输出不包括变更请求。
**【答案】(1) × (2) √ (3) ×**
> 解析：(1) 这是编码的定义。(2) 严格来说是更新干系人登记册（作为项目文件更新的一部分），但作为输出内容是正确的。(3) 监督沟通的输出包含变更请求。
