## 16.1 管理基础

### 16.1.1 协议/采购合同
*   **定义**：项目采购管理包括从项目团队外部采购或获取所需产品、服务或成果的各个过程。
*   **形式**：协议可以是合同、服务水平协议 (SLA)、谅解备忘录、协议备忘录 (MOA) 或订购单。
*   **法律效力**：合同中未规定的任何事项则不具法律强制力。无论合同规定如何详尽，文化和当地法律对合同及其可执行性均有影响。
*   **角色**：
    *   卖方：承包商、供货商、服务提供商。
    *   买方：最终产品所有人、分包商、收购机构。

### 16.1.2 管理新实践
1.  **工具的改进**：在线工具（广告、查找、填写）、建筑信息模型 (BIM)。
2.  **更先进的风险管理**：将具体风险分配给最有能力管理的一方。
3.  **变化中的合同签署实践**：重视密切合作（如折扣优惠），使用国际公认的标准合同范本。
4.  **物流和供应链管理**：明确主要及备选采购渠道。
5.  **技术和干系人关系**：使用网络摄像等技术改善沟通，减少现场争议。
6.  **试用采购**：在大批量采购前，先试用多个候选卖方。

## 16.3 规划采购管理

### 一般采购步骤
1.  准备采购工作说明书 (SOW) 或工作大纲 (TOR)。
2.  准备高层级的成本估算，制定预算。
3.  发布招标广告。
4.  确定合格卖方的名单。
5.  准备并发布招标文件。
6.  由卖方准备并提交建议书。
7.  对建议书开展技术（包括质量）评估。
8.  对建议书开展成本评估。
9.  准备最终的综合评估报告，选出中标建议书。
10. 结束谈判，签署合同。


![](images/pic_004.jpg)

### 16.3.1 输入
1.  **立项管理文件**。
2.  **项目章程**。
3.  **项目管理计划**（范围、质量、资源管理计划及范围基准）。
4.  **项目文件**：
    *   **风险登记册**：列明可转移给第三方的风险。
    *   **干系人登记册**：包括合同签署人员和法务人员。
    *   **需求文件**：技术要求和法律需求。
    *   **里程碑清单**：卖方交付成果的时间。
    *   **项目团队派工单**：评估团队是否具备采购能力。
5.  **事业环境因素**。
6.  **组织过程资产**。

### 16.3.2 工具与技术
1.  **专家判断**。
2.  **数据收集**：市场调研（考察行业和卖方能力）。
3.  **数据分析**：
    *   **自制或外购分析**：决定自行完成还是外部采购。考虑因素包括资源配置、技能需求、风险等。
4.  **供方选择分析**（常用方法）：
    *   **最低成本**：适用于标准化或常规采购。
    *   **仅凭资质**：采购价值小。
    *   **基于质量或技术方案**：先打技术分，入围后再谈价格。
    *   **基于质量和成本**：综合考虑，风险大时质量关键。
    *   **唯一来源**：特殊情况，无竞争。
    *   **固定预算**：在预算内选择技术得分最高的卖方。
5.  **会议**。



### 16.3.3 输出
1.  **采购管理计划**：
    *   包含协调工作、时间表、测量指标、干系人职责、制约因素、司法管辖权、独立估算需求、风险管理、预审合格卖方等。
2.  **采购策略**：
    *   交付方法、合同支付类型、采购阶段。
3.  **采购工作说明书 (SOW)**：
    *   依据范围基准编制。内容包括规格、数量、质量、绩效、履约期间、地点等。服务类采购称为“工作大纲 (TOR)”。
4.  **招标文件**：
    *   **信息邀请书 (RFI)**：征求更多信息。
    *   **报价邀请书 (RFQ)**：征求价格/成本信息。
    *   **建议邀请书 (RFP)**：解决办法难以确定，征求方案。
5.  **自制或外购决策**。
6.  **独立成本估算**：作为评价卖方报价的基准（标底）。若差异大，可能SOW有缺陷或卖方误解。
7.  **供方选择标准**：确保选出最佳质量的建议书。
8.  **变更请求**。

**采购文件内容对比表：**

| 采购管理计划 | 采购策略 | 工作说明书 (SOW) | 招标文件 |
| :--- | :--- | :--- | :--- |
| 协调与整合（资源/进度/预算） | 采购交付方法 | 采购项目描述 | 信息邀请书 (RFI) |
| 关键活动时间表 | 协议类型 | 规格、质量、绩效 | 报价邀请书 (RFQ) |
| 采购指标、职责 | 采购阶段 | 附加服务描述 | 建议邀请书 (RFP) |
| 假设、制约、法律、货币 | | 验收标准、担保 | |

## 16.4 实施采购

**核心作用**：选定合格卖方并签署法律协议。

### 16.4.1 输入
1.  项目管理计划。
2.  项目文件。
3.  **采购文档**：招标文件、SOW、独立成本估算、供方选择标准。
4.  **卖方建议书**：建议要求价格与技术分开。
5.  EEF & OPA。

![](images/pic_010.jpg)


### 16.4.2 工具与技术
1.  **专家判断**。
2.  **广告**。
3.  **投标人会议**（标前会议）：
    *   目的：确保所有潜在投标人对要求有清楚且一致的理解。
    *   原则：没有任何投标人得到特别优待。
4.  **数据分析**（建议书评估）：确定是否响应了招标文件。
5.  **人际关系与团队技能**（谈判）：
    *   由拥有合同签署职权的成员主导。项目经理提供协助。



### 16.4.3 输出
1.  **选定的卖方**。
2.  **协议 (合同)**：
    *   对双方都有约束力。
    *   内容包括：SOW、进度、绩效报告、支付条款、检查验收、担保、保险、变更处理、争议解决等。
3.  变更请求。

## 16.5 控制采购

**核心**：管理合同关系，确保买卖双方履行义务。

### 16.5.1 输入
1.  项目管理计划。
2.  项目文件。
3.  **采购文档**（完整支持性记录）。
4.  协议。
5.  工作绩效数据。
6.  批准的变更请求。

![](images/pic_013.jpg)


### 16.5.2 工具与技术
1.  **专家判断**。
2.  **索赔管理**：
    *   争议解决优先顺序：**谈判 (协商)** > 调解 > 仲裁 > 诉讼。
3.  **数据分析**：
    *   **绩效审查**：对照协议审查质量、进度、成本。
    *   **挣值分析 (EVA)**：计算偏差 (CV/SV) 和指数 (CPI/SPI)。
    *   **趋势分析**：预测完工估算 (EAC)。
4.  **检查**：对**承包商工作**的结构化审查（实地审查或成果审查）。
5.  **审计**：对**采购过程**的结构化审查（关注过程本身）。



### 16.5.3 输出
1.  **采购关闭**：
    *   发出正式书面通知。
    *   要求：按时按质交付、无未决索赔、款项付清、成果获批。
2.  采购文档 (更新)。
3.  工作绩效信息。

## 16.6 项目合同管理

### 16.6.1 合同的类型 (重点)

#### 1. 按项目范围划分
*   **总承包合同**：业主只与一个卖方签约。适用于经验丰富、实力强的卖方。
*   **单项承包合同**：业主分别与不同卖方签约。对买方协调能力要求高。
*   **分包合同**：
    *   条件：买方认可、非主体工作、不能转包整个项目、分包方有资质、不能再次分包。
    *   责任：买方可追究卖方或分包方责任。

#### 2. 按项目付款方式划分 (核心考点)

| 大类 | 具体类型 | 描述与适用场景 | 风险承担 |
| :--- | :--- | :--- | :--- |
| **总价合同**<br>(Fixed Price) | **固定总价 (FFP)** | 价格固定，除非范围变更。<br>**适用**：范围非常明确。<br>**特点**：买方最喜欢，卖方承担成本风险。 | 卖方风险最大 |
| | **总价加激励费用 (FPIF)** | 允许绩效偏离，设置财务奖励。需设置**价格上限**。<br>**特点**：高于上限的成本由卖方承担。 | |
| | **总价加经济价格调整 (FP-EPA)** | 履约周期长，允许根据通胀/降价调整。 | |
| | **订购单 (单边合同)** | 非大量采购标准化产品。 | |
| **成本补偿合同**<br>(Cost Reimbursable) | **成本加固定费用 (CPFF)** | 成本实报实销 + 一笔固定费用（利润）。<br>**适用**：范围不清楚。<br>**风险**：买方承担成本风险。 | 买方风险最大 |
| | **成本加激励费用 (CPIF)** | 成本实报实销 + 激励费用。<br>**计算**：根据成本节约或超支，按比例分摊。<br>公式：总付款 = 目标成本 + 目标费用 + (实际成本 - 目标成本) $\times$ 买方分担比例 | |
| | **成本加奖励费用 (CPAF)** | 成本实报实销 + 奖励费用。<br>**特点**：奖励由买方主观判断，卖方无权申诉。 | |
| **工料合同**<br>(T&M) | **工料合同** | 按实际工时和材料单价付费。<br>**适用**：工作性质清楚但**工作量不确定**；需快速签订合同。<br>**特点**：混合型（兼具成本补偿和总价特点）。 | 买方承担工作量风险<br>卖方承担单价风险 |

**合同类型选择指南：**
*   范围明确 $\rightarrow$ **总价合同**。
*   范围不清楚 $\rightarrow$ **成本补偿合同**。
*   性质清楚但量不确定 / 需快速签约 $\rightarrow$ **工料合同**。
*   双方分担风险 $\rightarrow$ **工料合同** / **激励费用类**。



### 16.6.2 合同的内容
包括项目名称、范围、质量、进度、保密、风险、成果归属、验收标准、价款支付、违约金、争议解决、术语解释等。



### 16.6.3 合同管理过程

1.  **合同的签订管理**
    *   建议使用标准格式。
    *   标的描述准确（忌“计算机一台”）。
    *   明确质量、变更、转让条款。
    *   **解释顺序**：如果有多个文件，需列明适用顺序。
    *   公证可增强法律效力。

2.  **合同的履行管理**
    *   争议解决：谈判（协商）$\rightarrow$ 调解 $\rightarrow$ 仲裁 $\rightarrow$ 诉讼。

3.  **合同的变更管理**
    *   书面申请 $\rightarrow$ 书面答复 $\rightarrow$ 生效。

4.  **合同的档案管理**
    *   文本管理是基础，一般要求电脑打印，手写旁注无效。

5.  **合同违约索赔管理**
    *   **分类**：工期索赔、费用索赔。
    *   **原则**：客观原因（如反常天气）可索赔工期，一般无费用；买方原因造成拖延，可索赔工期 + 费用。
    *   **依据**：有理性、有效性、正确性。
    *   **时限**：通常为 **28天**（提出索赔、报送资料、监理答复均为28天）。

**合同解释原则：**
1.  **主导语言原则**：以主导语言为准。
2.  **适用法律原则**：依据规定的法律解释。
3.  **整体解释原则**：
    *   特殊优先于一般。
    *   具体优先于笼统。
    *   **手写优先于印刷**。
    *   **单价优先于总价**。
    *   文字优先于数字。
    *   **技术规范优先于图纸**。
4.  **公平诚信原则**：若仍不清，按**不利于起草方**（通常为买方）原则解释。

## 本章练习与真题

**练习题：**

(1) 在采购规划过程中，需要考虑组织过程资产等一系列因素，（）不是采购规划时需要考虑的。
A. 项目管理计划
B. 风险登记册
C. 采购工作说明书
D. 干系人登记册
**【答案】C**
> 解析：采购工作说明书 (SOW) 是规划采购管理的**输出**，不是输入。

(2) 在确定项目合同类型时，如果项目工作范围很明确且风险不大，建议使用 ( )。

(4) 在 CPIF 合同下，A 公司是卖方，B 公司是买方，合同的实际成本大于目标成本时，A 公司得到的付款总数是（）。
A. 目标成本 + 目标费用 - B 公司应担负的成本超支
B. 目标成本 + 目标费用 + A 公司应担负的成本超支
C. 目标成本 + 目标费用 - A 公司应担负的成本超支
D. 目标成本 + 目标费用 + B 公司应担负的成本超支
**【答案】D**
> 解析：虽然 C 和 D 在数学逻辑上可能指向同一结果（视分担比例正负号定义），但标准表述是：A公司（卖方）得到的付款 = 实际成本 + (目标费用 - 卖方分担的超支)。
> 或者理解为：目标成本 + 目标费用 + (买方分担的超支)。如果实际成本 > 目标成本，说明超支，卖方不仅拿不到激励，还要倒扣（分担超支）。
> 本题D选项表述较为绕口，通常公式为：付款 = 实际成本 + 目标费用 + (目标成本 - 实际成本) $\times$ 卖方分担比例。
> 如果按原文截图逻辑：实际成本 > 目标成本时，卖方得到 = 目标成本 + 目标费用 + **买方应负担的成本超支**。这是因为总支付 = 实际成本（即目标成本+超支）+ 调整后的费用。这是一个特定视角的表述。
